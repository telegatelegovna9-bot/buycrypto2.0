# Binance SL/TP API Fix - Обновленная версия

## Что изменено в этот раз

### 1. Улучшенное логирование
Добавлено детальное логирование для каждой попытки выставления SL/TP:

**Для Stop-Loss:**
- Символ (оригинальный и нормализованный)
- Side (BUY/SELL)
- Stop Price
- Quantity
- Полный текст ошибки от Binance

**Для Take-Profit:**
- Аналогичная структура логирования
- TP Price вместо Stop Price

### 2. Формат логов ошибок
```
[NATIVE API] SL Request - Symbol: BTCUSDT, Type: STOP_MARKET, Side: SELL, StopPrice: 95000.0, Quantity: 0.001, ReduceOnly: True
[NATIVE API] SL order FAILED: <текст ошибки>
[NATIVE API] SL ERROR CODE DETECTED: <код ошибки если есть>
[NATIVE API] SL FULL ERROR DETAILS:
  - Symbol: BTC/USDT:USDT (normalized: BTCUSDT)
  - Side: SELL
  - Stop Price: 95000.0
  - Quantity: 0.001
  - Error Message: <полная ошибка>
```

## Как протестировать

### Шаг 1: Запустите бота
```bash
cd /workspace/buycrypto2.0
python main.py
```

### Шаг 2: Дождитесь открытия позиции
Бот откроет позицию согласно сигналам стратегии.

### Шаг 3: Проверьте логи
Смотрите файл `/workspace/buycrypto2.0/logs/trading_bot.log` или вывод в консоли.

Ищите сообщения:
- `[NATIVE API] Placing SL via /fapi/v1/order`
- `[NATIVE API] Placing TP via /fapi/v1/order`
- `[NATIVE API] SL order placed SUCCESS` или `[NATIVE API] SL order FAILED`
- `[NATIVE API] TP order placed SUCCESS` или `[NATIVE API] TP order FAILED`

### Шаг 4: Если есть ошибки
Скопируйте **ПОЛНЫЙ** блок ошибки включая:
1. Строку с параметрами запроса
2. Строку с кодом ошибки (если есть)
3. Все строки `FULL ERROR DETAILS`

## Возможные ошибки и решения

### Ошибка: "Invalid stop price"
**Причина:** Цена стопа вне допустимого диапазона
**Решение:** Проверить цену относительно текущей рыночной

### Ошибка: "Quantity precision error"
**Причина:** Количество контрактов не соответствует точности биржи
**Решение:** Будет добавлена автоматическая коррекция

### Ошибка: "ReduceOnly order rejected"
**Причина:** Нет открытой позиции или позиция уже закрыта
**Решение:** Проверить наличие активной позиции

### Ошибка: "Timestamp ahead of server time"
**Причина:** Рассинхронизация времени
**Решение:** Автоматическая синхронизация при старте

### Ошибка: "Invalid symbol"
**Причина:** Неправильная нормализация символа
**Решение:** Проверить формат символа в логах
