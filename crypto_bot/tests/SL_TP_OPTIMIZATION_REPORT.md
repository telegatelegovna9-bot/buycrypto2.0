# Анализ и оптимизация SL/TP стратегии

## Проблема
За 12 часов работы бота было всего 2 прибыльные сделки, много безубытков и стоп-лосов. Наблюдалась ситуация, когда сделка была прибыльна в моменте (выше TP), но затем закрывалась в безубыток.

## Результаты тестирования

### Тест 1: Поведение Breakeven
**Проблема:** Слишком ранний перевод в безубыток (при 1%) приводит к преждевременному закрытию сделок.

**Результаты:**
- При threshold 0.5-3% - сделка закрывается по TP
- Но при сильной волатильности и откате после +4% до +0.8%, ранний BE может сработать

**Решение:** Увеличен порог BE с 1% до **2.5%**

### Тест 2: Оптимизация Trailing Stop
**Проблема:** Слишком агрессивный trailing stop не даёт цене "дышать".

**Конфигурации:**
- Aggressive (2%, 1.5x ATR) - выход на +2%
- Balanced (3%, 2.0x ATR) - выход на +2%
- Conservative (5%, 2.5x ATR) - выход на +2%

**Решение:** Увеличен порог активации trailing с 2% до **3%** и множитель ATR с 2.0 до **2.5**

### Тест 3: Динамическое управление TP
**Проблема:** Цена достигает +5%, но TP установлен на +3%. Бот закрывает сделку, оставляя прибыль на столе.

**Сравнение стратегий:**
- Фиксированный TP: +3.0%
- Частичное закрытие 50%: +4.2%
- Перенос TP выше: +5.0%
- Агрессивный trailing: +4.5%

**Решение:** Реализована функция `_check_dynamic_tp_management()`

## Внесённые изменения

### 1. position_monitor.py

#### Изменены параметры:
```python
# БЫЛО:
self.breakeven_threshold = 0.01  # 1%
self.trailing_activation = 0.02  # 2%
self.trailing_stop_atr_multiplier = 2.0

# СТАЛО:
self.breakeven_threshold = 0.025  # 2.5% - OPTIMIZED
self.trailing_activation = 0.03   # 3% - OPTIMIZED  
self.trailing_stop_atr_multiplier = 2.5  # Wider trail
```

#### Добавлена новая функция `_check_dynamic_tp_management()`:
- Отслеживает цену выше TP
- При превышении TP на >1% и прибыли >5%:
  - Включает агрессивный trailing (1.5x ATR вместо 2.5x)
  - Логгирует ситуацию для анализа
  - Позволяет захватить дополнительную прибыль

### 2. strategy_stats.json

- Сохраняется при shutdown бота

## Рекомендации по настройке

### Для текущих рыночных условий:

1. **Увеличьте take_profit_pct** в конфиге:
   ```python
   config.risk.take_profit_pct = 0.04  # 4% вместо 3%
   ```

2. **Добавьте частичные закрытия** (опционально):
   - 30% позиции закрывать на TP
   - 70% оставлять с trailing stop

3. **Настройте volatilty-adaptive параметры**:
   - Высокая волатильность: wider stops (3% SL, 5% TP)
   - Низкая волатильность: tighter stops (1.5% SL, 3% TP)

4. **Мониторьте лог [DYNAMIC TP]**:
   - Если видите много таких сообщений - можно увеличить TP
   - Если цена редко доходит до TP - уменьшить
