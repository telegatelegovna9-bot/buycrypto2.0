# 🧠 УМНЫЙ МОНИТОРИНГ TP/SL С ИНДИКАТОРАМИ

## Проблема
Ранее бот сталкивался с проблемой: **монета доходила до TP, но бот не закрывал позицию**, а затем переставлял TP и SL. В итоге монета разворачивалась и бот забирал только 30% движения вместо 100%, а иногда даже уходил в стоп.

## Решение
Реализована **интеллектуальная система управления позицией** на основе индикаторов в реальном времени.

---

## 🔑 Ключевые изменения

### 1. МГНОВЕННАЯ ЗАЩИТА ПРИБЫЛИ (TP PROTECTION)
Когда цена достигает TP:
- **СРАЗУ** ставим SL на уровень TP (минус комиссия)
- Это гарантирует что мы не потеряем прибыль при откате
- Логирование: `[TP PROTECTION]`

```python
# Было: бот ждал и надеялся
# Стало: сразу защищаем прибыль
if current_price >= take_profit:
    stop_loss = take_profit * 0.999  # На уровне TP
```

### 2. АНАЛИЗ ИНДИКАТОРОВ В РЕАЛЬНОМ ВРЕМЕНИ
При достижении TP бот анализирует:

| Индикатор | Что показывает | Влияние на решение |
|-----------|---------------|-------------------|
| **RSI (14)** | Перекупленность/перепроданность | RSI > 75 → сигнал на закрытие |
| **MACD (12,26,9)** | Направление тренда | MACD < 0 → медвежий сигнал |
| **Volume Ratio** | Сила движения | Volume > 2x → поддержка тренда |
| **Trend Strength** | Отклонение от EMA20 | Trend > 2% → сильный тренд |
| **ATR** | Волатильность | Для расчета trailing stop |

### 3. СИСТЕМА ПРИНЯТИЯ РЕШЕНИЙ (SCORING)
Каждый индикатор дает очки:
- **Положительные очки** → держать/двигать TP дальше
- **Отрицательные очки** → закрывать позицию

#### Пример анализа для LONG позиции:
```
RSI = 78  → -2 очка (перекупленность)
MACD < 0  → -2 очка (медвежий)
Volume = 2.5x → +2 очка (высокий объем)
Trend = +3% → +2 очка (сильный тренд)
─────────────────────────────────────
Итого: 0 очков → ДЕРЖАТЬ с trailing stop
```

### 4. 4 ТИПА РЕШЕНИЙ

| Решение | Когда применяется | Действие |
|---------|------------------|----------|
| **CLOSE_ALL** | Score ≤ -3 | Закрыть всю позицию |
| **CLOSE_PARTIAL** | Score ≤ -1 | Закрыть 50% позиции |
| **MOVE_TP** | Score ≥ 3 + excess > 2% | Двинуть TP дальше (цена + 2×ATR) |
| **HOLD** | Score ≥ 0 | Держать с trailing stop |

## 📊 Как это работает на практике

### Сценарий 1: Монета достигла TP, тренд сильный
```
[TP PROTECTION] BTC/USDT: SL переставлен на уровень TP! 42100 -> 42950 (прибыль защищена)
[TP ANALYSIS] BTC/USDT | Score: +4 | Action: MOVE_TP | Confidence: 80% | 
  MACD сильный бычий | Объем высокий (2.3x) | Тренд сильный (+2.5%)
[TP DECISION] BTC/USDT: ДВИНУТЬ TP | 43000 -> 43850 | Причина: Сильный тренд, двигаем TP
```
**Результат**: Бот защитил прибыль и дал позиции идти дальше → **+15% к изначальной цели**

### Сценарий 2: Монета достигла TP, но слабеет
```
[TP PROTECTION] ETH/USDT: SL переставлен на уровень TP! 2250 -> 2298 (прибыль защищена)
[TP ANALYSIS] ETH/USDT | Score: -4 | Action: CLOSE_ALL | Confidence: 90% |
  RSI перекупленность (79.2) | MACD медвежий | Тренд слабый (-1.2%)
[TP DECISION] ETH/USDT: ЗАКРЫТЬ ВСЮ ПОЗИЦИЮ | 
  Причина: Сильные сигналы на выход | PnL: +5.8%
```
**Результат**: Бот закрыл позицию на пике → **забрал 100% прибыли вместо 30%**

### Сценарий 3: Неопределенность
```
[TP PROTECTION] SOL/USDT: SL переставлен на уровень TP! 98.5 -> 102.3 (прибыль защищена)
[TP ANALYSIS] SOL/USDT | Score: +1 | Action: HOLD | Confidence: 60% |
  RSI нейтральный (58.3) | MACD нейтральный | Объем выше среднего (1.6x)
[TP DECISION] SOL/USDT: ДЕРЖАТЬ | PnL: +4.2% | Причина: Нейтральные/положительные сигналы
```
**Результат**: Бот держит позицию с trailing stop → **позволяет взять больше если пойдет дальше**

## 🛠 Технические детали

### Новые методы в `position_monitor.py`:

1. **`_secure_profit_at_tp()`** - Мгновенная защита прибыли
2. **`_fetch_realtime_indicators()`** - Получение индикаторов (RSI, MACD, Volume, ATR)
3. **`_calculate_rsi()`** - Расчет RSI
4. **`_calculate_macd()`** - Расчет MACD
5. **`_calculate_atr()`** - Расчет ATR
6. **`_analyze_exit_decision()`** - Анализ и принятие решения
7. **`_execute_exit_decision()`** - Выполнение решения

### Частота анализа:
- Проверка каждые **1 секунду**
- Индикаторы обновляются на каждом цикле
- Решение принимается мгновенно при достижении TP

## 🎯 Рекомендации по настройке

### Для агрессивной торговли:
```python
# В config/settings.py
breakeven_threshold = 0.02      # 2%
trailing_activation = 0.025     # 2.5%
# Бот будет быстрее фиксировать прибыль
```

### Для консервативной торговли:
```python
breakeven_threshold = 0.03      # 3%
trailing_activation = 0.04      # 4%
# Бот даст позиции больше пространства
```

### Пороги для scoring системы:
```python
# В _analyze_exit_decision():
if score <= -3:    # Более чувствительный выход
if score <= -1:    # Частичная фиксация
if score >= 3:     # Двигать TP дальше
```
