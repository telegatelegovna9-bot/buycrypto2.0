## 🔷 ОБЩАЯ АРХИТЕКТУРА

Бот представляет собой модульную систему для автоматической торговли фьючерсами на Binance.

### Основные компоненты:
- **main.py**: Центральный контроллер, координирующий все процессы
- **meta_controller.py**: Управляет стратегиями, определяет рыночный режим, агрегирует сигналы
- **position_monitor.py**: Мониторинг открытых позиций в реальном времени (каждую секунду)
- **execution_engine.py**: Исполнение ордеров на бирже (Binance Futures)
- **risk_manager.py**: Управление рисками, расчет размеров позиций, хранение активных позиций
- **data/data_loader.py**: Загрузка данных, скрининг пар, расчет индикаторов
- **strategies/**: 8 торговых стратегий
- **utils/telegram_notifier.py**: Отправка уведомлений в Telegram

### Цикл работы:
1. Скрининг рынка каждые N секунд
2. Определение рыночного режима (TREND_UP, TREND_DOWN, SIDEWAYS, VOLATILE)
3. Генерация сигналов от всех 8 стратегий
4. Агрегация сигналов и фильтрация по уверенности
5. Проверка риск-менеджмента (лимиты, маржа, корреляция)
6. Открытие позиции с расчетом SL/TP
7. Непрерывный мониторинг позиции (каждую секунду)
8. Динамическое управление SL/TP или закрытие по индикаторам
9. Фиксация результата, обновление статистики, уведомление

## 🎯 СТРАТЕГИИ ТОРГОВЛИ (8 ШТУК)

Все стратегии наследуются от `BaseStrategy` и возвращают сигнал: `long`, `short` или `None` с уровнем уверенности (0.0–1.0).

### 1. TrendBreakoutStrategy (Пробой тренда)
**Логика:**
- Определяет локальный тренд по скользящим средним (EMA 9, 21, 50)
- Ищет пробой ключевых уровней поддержки/сопротивления
- Вход при подтверждении пробоя объемом

**Условия входа LONG:**
- EMA(9) > EMA(21) > EMA(50) — восходящий тренд
- Цена пробивает сопротивление (high за N свечей)
- Объем выше среднего за период
- RSI < 70 (не перекупленность)

**Условия входа SHORT:**
- EMA(9) < EMA(21) < EMA(50) — нисходящий тренд
- Цена пробивает поддержку (low за N свечей)
- Объем выше среднего
- RSI > 30 (не перепроданность)

**Сила сигнала:** Зависит от угла наклона EMA, силы пробоя, объема.

---

### 2. VolatilityBreakoutStrategy (Пробой волатильности)
**Логика:**
- Использует полосы Боллинджера и ATR (Average True Range)
- Ищет сжатие волатильности с последующим резким расширением
- Вход при выходе цены за пределы полос с высоким объемом

**Условия входа LONG:**
- Цена закрывается выше верхней полосы Боллинджера
- ATR растет (волатильность расширяется)
- Объем выше среднего на 50%+
- Предыдущие свечи показывали сжатие (узкие полосы)

**Условия входа SHORT:**
- Цена закрывается ниже нижней полосы Боллинджера
- ATR растет
- Объем выше среднего
- Предыдущее сжатие волатильности

**Особенность:** Хорошо работает на новостях и резких движениях.

---

### 3. VolumeSpikeStrategy (Всплеск объема)
**Логика:**
- Детектирует аномальные всплески объема (в 2-5 раз выше среднего)
- Анализирует направление движения цены при всплеске
- Вход в направлении импульса

**Условия входа LONG:**
- Объем > 2.5 × средний объем за 20 свечей
- Цена растет на всплеске объема (зеленая свеча)
- Close > Open на значительную величину
- RSI не в зоне перекупленности (>80)

**Условия входа SHORT:**
- Объем > 2.5 × средний объем
- Цена падает на всплеске (красная свеча)
- Close < Open значительно
- RSI не в зоне перепроданности (<20)

**Фильтр:** Игнорирует всплески на боковике без направления.

---

### 4. LiquidityGrabStrategy (Захват ликвидности)
**Логика:**
- Ищет ложные пробои уровней (ликвидность стопов)
- Определяет зоны скопления стоп-ордеров (выше high / ниже low)
- Вход против ложного пробоя с быстрым разворотом

**Условия входа LONG:**
- Цена делает новый локальный минимум (ниже предыдущего low)
- Быстрый отскок вверх в пределах 1-3 свечей
- Формируется длинная нижняя тень (пин-бар)
- Объем на отскоке выше, чем на падении
- RSI показывает дивергенцию (цена ниже, RSI выше)

**Условия входа SHORT:**
- Цена делает новый локальный максимум
- Быстрый отскок вниз
- Длинная верхняя тень
- Объем на отскоке вниз высокий
- Медвежья дивергенция RSI

**Особенность:** Стратегия контртрендовая, ловит "охоту за стопами".

---

### 5. MeanReversionStrategy (Возврат к среднему)
**Логика:**
- Основана на теории, что цена стремится вернуться к скользящей средней
- Работает в боковом рынке (определяется MetaController)
- Использует RSI и полосы Боллинджера для определения экстремумов

**Условия входа LONG:**
- Рыночный режим = SIDEWAYS
- RSI < 30 (перепроданность)
- Цена касается или пересекает нижнюю полосу Боллинджера
- Следующая свеча закрывается выше минимума

**Условия входа SHORT:**
- Рыночный режим = SIDEWAYS
- RSI > 70 (перекупленность)
- Цена касается верхней полосы Боллинджера
- Следующая свеча закрывается ниже максимума

**Выход:** При возврате цены к средней линии (EMA 20) или противоположной полосе.

---

### 6. MomentumStrategy (Моментум)
**Логика:**
- Торговля по силе импульса
- Использует MACD, ROC (Rate of Change), силу тренда
- Вход при усилении импульса

**Условия входа LONG:**
- MACD гистограмма растет и выше 0 (или пересекает 0 снизу)
- ROC(14) > 0 и растет
- Цена выше EMA(21)
- Объем поддерживается или растет

**Условия входа SHORT:**
- MACD гистограмма падает и ниже 0 (или пересекает 0 сверху)
- ROC(14) < 0 и падает
- Цена ниже EMA(21)
- Объем на движении вниз

**Особенность:** Выход при ослаблении моментума (MACD разворот).

---

### 7. VolumeStrategy (Объемный анализ) — из volume_oi_strategies.py
**Логика:**
- Глубокий анализ объема: кумулятивный объем, VWAP, профиль объема
- Определяет зоны интереса крупных игроков

**Условия входа LONG:**
- Цена выше VWAP (Volume Weighted Average Price)
- Кумулятивный объем дельта положительный (покупок больше)
- Пробой уровня сопротивления с объемом выше POC (Point of Control)
- Подтверждение ростом открытого интереса (если доступно)

**Условия входа SHORT:**
- Цена ниже VWAP
- Кумулятивная дельта отрицательная
- Пробой поддержки с высоким объемом продаж
- Рост OI на падении (новые шорты)

**Индикаторы:** VWAP, Cumulative Volume Delta, Volume Profile, POC.

---

### 8. OpenInterestStrategy (Анализ открытого интереса) — из volume_oi_strategies.py
**Логика:**
- Анализирует изменения Open Interest (OI) вместе с ценой
- Определяет, кто доминирует: лонгисты или шортисты
- Выявляет потенциальные сквизы (long/short squeeze)

**Условия входа LONG:**
- Цена растет + OI растет → новые лонги, сильный тренд
- Цена падает + OI падает → закрытие шортов, возможен разворот вверх
- Резкий рост OI после консолидации → подготовка к движению

**Условия входа SHORT:**
- Цена падает + OI растет → новые шорты, сильный нисходящий тренд
- Цена растет + OI падает → закрытие лонгов, возможен разворот вниз
- Экстремально высокий OI + разворот цены → сквиз

**Фильтры:**
- Игнорирует низкий OI (недостаточно ликвидности)
- Учитывает funding rate (если высокий положительный → риск long squeeze)

## 🛡️ СИСТЕМА УПРАВЛЕНИЯ ПОЗИЦИЯМИ (TP/SL/MONITORING)

### 1. Первичная установка SL/TP при открытии

При открытии позиции бот рассчитывает:

**Stop Loss (SL):**
- На основе ATR (Average True Range): `SL = Entry ± (ATR × множитель)`
- Множитель зависит от волатильности пары (обычно 1.5–2.5 ATR)
- Для трендовых стратегий: дальше от входа
- Для mean reversion: ближе к входу

**Take Profit (TP):**
- Минимум 2:1 Reward/Risk ratio
- `TP = Entry ± (ATR × множитель × 2+)`
- Может быть несколько уровней TP (частичная фиксация)
- Динамический расчет на основе ближайших уровней сопротивления/поддержки

**Пример:**
```
Entry: $100
ATR: $2
SL: $100 - (2 × 1.5) = $97 (риск $3)
TP: $100 + (2 × 3) = $106 (профит $6, R/R = 2:1)
```

### 2. PositionMonitor — непрерывный мониторинг (каждую секунду)

Модуль `position_monitor.py` работает в фоне и выполняет:

#### A. Проверка достижения SL/TP
- Сравнивает текущую цену с уровнями SL/TP каждой позиции
- При достижении → немедленное закрытие по рынку
- Обновление статистики и отправка уведомления

#### B. Trailing Stop (Трейлинг-стоп)
- **Активируется** при достижении 50% пути к TP
- **Логика:**
  - Если цена идет в прибыль, SL подтягивается вслед за ценой
  - Отставание = 1 ATR или фиксированный %
  - Никогда не двигается в обратную сторону (только в плюс)
  
**Пример:**
```
Entry: $100, SL: $97, TP: $106
Цена достигла $103 (50% TP) → активируется трейлинг
Цена выросла до $104 → SL подтягивается до $102
Цена упала до $102 → срабатывает SL, фиксация прибыли $2
```

#### C. Breakeven Stop (Перевод в безубыток)
- **Активируется** при достижении 30-40% пути к TP
- **Действие:** SL перемещается на уровень входа (или вход + комиссия)
- Гарантирует отсутствие убытка при развороте

#### D. Dynamic TP Extension (Расширение TP)
- Если цена достигает TP и тренд сильный (по индикаторам):
  - TP автоматически сдвигается выше
  - Фиксируется частичная прибыль (50% позиции)
  - Оставшаяся часть идет дальше с трейлинг-стопом

#### E. Early Exit по индикаторам (Умный выход)
Если сделка в плюсе (+30%, +50%, +75%), но индикаторы показывают слабость:

**Условия досрочного закрытия LONG:**
- RSI входит в зону перекупленности (>75) и разворачивается вниз
- MACD гистограмма начинает падать (дивергенция)
- Объем падает на попытке роста (слабость покупателей)
- Цена теряет ключевой уровень поддержки

**Условия досрочного закрытия SHORT:**
- RSI входит в перепроданность (<25) и разворачивается вверх
- MACD показывает бычье расхождение
- Объем падает на движении вниз
- Цена пробивает сопротивление вверх

**Решение:**
- Если PnL > 30% от целевого → закрытие всей позиции
- Если PnL > 50% → можно закрыть 50%, остальное с трейлингом
- Бот выбирает оптимальный момент, чтобы не отдать прибыль обратно

#### F. Синхронизация с биржей в реальном времени
Каждую секунду PositionMonitor:
1. Запрашивает актуальные позиции с Binance
2. Сравнивает со своим списком
3. **Если позиция исчезла с биржи** (закрыта вручную):
   - Локально закрывает позицию в risk_manager
   - Обновляет `strategy_stats.json`
   - Отправляет уведомление в Telegram: "Ручное закрытие (биржа)"
4. **Если появилась новая позиция на бирже** (после перезапуска):
   - Восстанавливает контроль над ней
   - Начинает мониторинг

## 📊 ИНДИКАТОРЫ И ТЕХНИЧЕСКИЙ АНАЛИЗ

### Используемые индикаторы:

| Индикатор | Описание | Где применяется |

| **EMA (9, 21, 50)** | Экспоненциальные скользящие средние | TrendBreakout, Momentum, определение тренда |

| **Volume** | Объем торгов | Все стратегии, подтверждение пробоев |
| **VWAP** | Средневзвешенная цена по объему | VolumeStrategy |
| **Cumulative Volume Delta** | Разница объемов покупок/продаж | VolumeStrategy |
| **Open Interest** | Открытый интерес фьючерсов | OpenInterestStrategy |

- Доступен для всех стратегий через общий интерфейс

## ⚖️ РИСК-МЕНЕДЖМЕНТ

### 1. Расчет размера позиции
- Фиксированный риск на сделку: 1-2% от баланса
- Формула: `Size = (Balance × Risk%) / (Entry - SL)`
- Учет левериджа (макс. 3-5x в зависимости от волатильности)

### 2. Лимиты
- Максимум одновременных позиций: 3
- Максимальный риск на все позиции: 5% от баланса
- Ограничение по секторам (не более 2 позиций из одной группы монет)

### 3. Маржа и ликвидация
- Проверка доступной маржи перед открытием
- Расчет цены ликвидации (должна быть далеко от SL)
- Автоматическое снижение левериджа при высокой волатильности

### 4. Корреляция
- Фильтрация высококоррелированных пар
- Не открывать одновременно BTC и ETH (высокая корреляция)
- Диверсификация по разным секторам рынка

### 5. Drawdown контроль
- При просадке >10% за день → снижение размера позиций в 2 раза
- При просадке >20% → остановка торговли до восстановления

## 🔄 СИНХРОНИЗАЦИЯ С БИРЖЕЙ

### Механизм синхронизации:

#### При старте бота (`sync_positions_from_exchange()`):
1. Запрос всех открытых позиций с Binance
2. Сравнение с локальным состоянием в risk_manager
3. **Восстановление позиций:**
   - Если позиция есть на бирже, но нет локально → добавить в мониторинг
   - Если размер отличается → обновить локальные данные
   - Если плечо отличается → использовать биржевые данные
4. Обновление баланса

#### В цикле мониторинга (каждую секунду):
1. Запрос актуальных позиций с биржи
2. Сравнение с локальным списком
3. **Обнаружение ручных закрытий:**
   - Позиция была локально, но исчезла с биржи
   - Закрытие в risk_manager с пометкой "manual_close"
   - Обновление статистики стратегии
   - Уведомление в Telegram
4. **Обнаружение новых позиций:**
   - Позиция появилась на бирже, но нет локально
   - Восстановление контроля, начало мониторинга

### Обработка ошибок:
- Если API Binance недоступен → использование последних известных данных
- Логирование всех расхождений
- Попытка повторной синхронизации через 5 секунд

- Потеря связи с биржей
- Недостаточно маржи
- Превышение лимита позиций
- Критические ошибки стратегий

## 📈 СТАТИСТИКА И АДАПТИВНОЕ ОБУЧЕНИЕ

### Файл `strategy_stats.json`:
Хранит статистику по каждой стратегии:

```json
{
  "TrendBreakout": {
    "total_trades": 15,
    "wins": 9,
    "losses": 6,
    "win_rate": 0.60,
    "total_pnl": 2.45,
    "avg_win": 0.42,
    "avg_loss": -0.18,
    "profit_factor": 2.33,
    "weight": 1.85
  },
  "Momentum": { ... },
  ...
}
```

### Адаптивные веса стратегий:
- Вес стратегии динамически меняется на основе:
  - Win Rate (чем выше, тем больше вес)
  - Profit Factor (отношение средней прибыли к среднему убытку)
  - Общего PnL
  - Недавних результатов (последние 10 сделок имеют больший вес)

### Влияние на сигналы:
- При агрегации сигналов от нескольких стратегий:
  - Сигнал умножается на вес стратегии
  - Стратегии с высоким весом имеют большее влияние
  - Низкий вес → сигнал может быть проигнорирован

### Обновление статистики:
- После каждого закрытия позиции (любой причины)
- Автосохранение в `strategy_stats.json`
- Пересчет весов каждые 5 сделок или при изменении win rate > 5%

## 🎛️ РЕЖИМЫ РАБОТЫ

### 1. Live Trading (Реальные деньги)
- Подключение к Binance Futures API
- Реальные ордера на покупку/продажу
- Реальный мониторинг позиций
- Отправка уведомлений в Telegram
- **Требует API ключи с правами на торговлю**

### 2. Backtesting (Тестирование на истории)
- Загрузка исторических данных
- Симуляция торговли без реальных ордеров
- Расчет метрик: Sharpe Ratio, Max Drawdown, Total Return
- Оптимизация параметров стратегий

### 3. Paper Trading (Демо-режим)
- Работа с реальными данными рынка
- Симуляция ордеров без реальных денег
- Полная функциональность как в Live режиме
- Идеально для тестирования новых стратегий

### Переключение режимов:
- Через конфиг файл `config.yaml`
- Параметр `trading_mode: live | backtest | paper`
- Разные наборы API ключей для каждого режима
